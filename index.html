The event loop is a series of queues - task queue and we can think of the rendering pipe-line as a kind of queue that only has one thing in it Describe how things get added to queues
Can have lots of other queues - Node has separate queues for timeouts and I/O callbacks Browsers might have separate queues for user-generated callbacks and DOM parsing callbacks
The event loop picks a queues, takes a task from it and runs it to completion, then picks another queue
Blocking event loop - add a task that takes a long time so other tasks can't run
More detail of Node queues and how the event loop picks things Difference between Node and browser - browser: can pick a task from any queue once this task finishes - Node: runs all the tasks in the current queue before picking another queue
In addition, there are a few special queues that Node and browsers implement Browsers: microtasks and requestAnimationCallback Node: process.nextTick & setImmediate
